## Known Ambiguities and Conservative Resolution Notes

1. Legacy status mismatch ‚Äî treat `SPEC-001` lifecycle fields as canonical; do not preserve legacy `status` semantics beyond temporary compatibility reads.
2. Namespace vocabulary resolution ‚Äî `sources` and `session` are canonical per `SPEC-004`; `exports` in `ARCHITECTURE.md` ¬ß18 is superseded.
3. `todos.move` return shape mismatch ‚Äî standardize on the `SPEC-004` response shape; do not support dual payload formats.
4. CLI flags not backed by service contracts ‚Äî expose only flags that current service boundaries can honor truthfully; unsupported flags should fail explicitly.
5. Watcher/SSE boot wiring missing ‚Äî land runtime event wiring before claiming realtime behavior in UI, CLI, or MCP.
6. `SPEC-010` stale against `SPEC-007`/`SPEC-008` ‚Äî when UI prose conflicts with later event/provenance contracts, prefer dependency order and update implementation docs accordingly.
7. Intelligence layer unimplemented ‚Äî keep `SPEC-009` off the critical path; do not invent placeholder analyzers or fake artifacts.
8. Source service boundary unresolved ‚Äî keep ingestion orchestration under review-owned services until `SPEC-006` defines a separate source boundary explicitly.
9. Persistence ownership sequencing tension ‚Äî `SPEC-005` should establish migration ownership and base review schema first; intelligence-specific DDL should be additive once `SPEC-008`/`SPEC-009` shapes are fixed.
10. UI route path inconsistencies ‚Äî treat the actual router tree under `src/routes` as canonical until `SPEC-010` is cut over.
